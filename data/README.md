# PoE2 Crafting Dataset (normalized)

Generated by `../pipeline/build_dataset.py` from [RePoE2](https://repoe-fork.github.io/poe2/)
(JSON exported from the 0.5 game files, client version 4.5.x). Re-run the pipeline to refresh.

## Files

- **`poe2_mods_by_class.json`** — the main artifact. Keyed by item class (`Sceptre`,
  `Body Armour`, `Ring`, ...). Each class holds its eligible `prefixes` and `suffixes`.
- **`poe2_bases.json`** — every released base item, grouped by item class
  (name, metadata id, tags, drop_level, implicits).
- **`poe2_meta.json`** — build timestamp, source, counts.

## Mod record schema

```jsonc
{
  "id": "IncreasedLife5",          // RePoE mod id
  "name": "Stalwart",              // affix name shown in-game
  "text": "+(40-59) to maximum Life",
  "side": "prefix",                // prefix | suffix
  "group": ["IncreasedLife"],      // exclusivity group: one mod per group per item
  "ilvl": 24,                       // item level required for THIS tier
  "tags": ["amulet","belt","body_armour","boots"],  // bases it can roll on
  "stat_ids": ["base_maximum_life"],
  "stats": [{"id":"base_maximum_life","min":40,"max":59}],
  "essence_only": false
}
```

## Two things to know when consuming this

1. **Class pool vs. specific base.** A class pool is the union over all bases in that
   class. For a *specific* base, filter mods to those whose `tags` intersect the base's
   `tags` (from `poe2_bases.json`). Example: an ES (`int_armour`) body armour only rolls
   the `int_armour`-tagged hybrid defence+life mods, not the `str_armour` ones.

2. **Odds are NOT real probabilities.** 0.5 game files ship spawn weights flattened to 1,
   so the only honest probability model is *uniform within the eligible pool*. Use mod
   counts as a grindiness signal ("N eligible suffixes"), not as true hit chances. The
   `group` field is what enforces "one mod per group" when reasoning about open slots.

## Not yet included (next pipeline passes)

Essence guaranteed-mods (cross-ref poe2db per base), Runic Alloys, desecrated/Lich mods
(`domain: desecrated` in mods.json), omen effects, and currency-step transition rules.
These feed the path planner's "what can place this mod" layer.
