# PoE2 Crafting Dataset (normalized)

Generated by `../pipeline/build_dataset.py` from [Craft of Exile](https://www.craftofexile.com/)'s
`poec_data.json` (cached at `../pipeline/cache/coe_poec_data.json`), plus essences from poe2db via
`../pipeline/build_essences.py`. Re-run the pipeline to refresh. (RePoE2 was the original backbone
but is retired — its 0.5 export flattens spawn weights to 0/1, so it can't give odds.)

## Files

- **`poe2_mods_by_class.json`** — the main artifact. Keyed by item class (`Sceptre`,
  `Body Armour`, `Ring`, ...). Each class holds `bases`, `prefixes`, `suffixes`, and the
  `desecrated` (Lich/exclusive) pool.
- **`poe2_bases.json`** — base items grouped by item class. Attribute/elemental variants
  (e.g. `Body Armour (STR)` vs `(INT)`, `Fire Wand` vs `Wand`) are SEPARATE bases with their
  own tags and weighted pools.
- **`poe2_essences.json`** — item-class-aware essence table (81 essences, 4 tiers) with the
  guaranteed mod per class and `mode` (`magic_to_rare` ADD vs Perfect `remove_add`).
- **`poe2_meta.json`** — build timestamp, source, counts.

## Mod record schema

```jsonc
{
  "id": "5056",                       // CoE modifier id
  "name": "+# to maximum Life",       // affix family name
  "text": "+(200-214) to maximum Life",
  "side": "prefix",                   // prefix | suffix
  "group": ["IncreasedLife"],         // exclusivity group: one mod per group per item
  "ilvl": 1,                           // min item level for the family's lowest tier
  "tags": ["Ring"],                    // exact base names it can roll on
  "mtags": ["Life"],                   // descriptive tags (Caster, Attack, Lich name, ...)
  "essence_only": false,
  "src": "base",                       // base | desecrated
  // per-base tier ladder: [ilvl, weight, tierText] — drives the picker AND the odds engine
  "bw": { "Ring": [[1, 1000, "+(10-19) to maximum Life"], [11, 1000, "+(20-29) ..."], ...] }
}
```

## Two things to know when consuming this

1. **Class pool vs. specific base.** A class pool is the union over all bases in that class.
   For a *specific* base, filter mods to those whose `tags` include the base's name. Each mod's
   weight on that base is in `bw[baseName]`.

2. **Odds are real (extrapolated) weights now.** `bw[base]` carries Craft of Exile's
   community-EXTRAPOLATED spawn weights per tier, so the planner reports honest weight-share odds
   ("~X% per slam, about 1 in N"). These are estimates, not official GGG numbers — treat them as
   solid *relative* effort, not exact probability. The `group` field still enforces "one mod per
   group" when reasoning about open slots.

## Included now

Essence guaranteed-mods (`poe2_essences.json`) and desecrated/Lich mods (the `desecrated` pool per
class, `src: "desecrated"`) are wired into the planner.
